## ✅ 强制性要求

### 2. 本地持久化文件夹命名规范

#### 2.1 文件夹命名

**必须遵守**：

```
output/results/{task_id}/
```

**路径位置说明**：
- 基准位置：**当前技能工作区根目录**（优先为同时包含 `BigPoi-verification-qc` 和 `qc-write-pg-qc` 的目录）
- 回退位置：若无法识别当前技能工作区，再回退到含有 `.claude` 或 `.openclaw` 的目录
- 例如：`D:\poi_qc\output\results\{task_id}\`
- result_persister.py 会自动从当前工作目录和脚本目录向上查找可用根目录

| 要求               | 说明                                             | 示例                                                                                  |
| ------------------ | ------------------------------------------------ | ------------------------------------------------------------------------------------- |
| **目录名**   | 必须使用 `task_id`，与上游数据完全一致，不做任何转换 | ✅`81B39041-0840-4C89-9F4B-075A80F2CDD4`（UUID格式）<br>✅`219A8C6D8C334629A7E1F164D514C381`（大写字母+数字） |
| **路径结构** | `output/results/{task_id}/`（相对于项目根目录）                    | `output/results/81B39041-0840-4C89-9F4B-075A80F2CDD4/`                                  |
| **创建时机** | 质检结果生成后，立即创建该目录                   | 不可延迟创建                                                                          |

**❌ 禁止的做法**：

```
output/results/id_5bcdc5b3/        ❌ 不可使用 id 作为目录名
output/results/5bcdc5b3/           ❌ 不可修改或截断 task_id
output/results/poi_456/            ❌ 不可使用其他业务ID
output/results/UPPERCASE/          ❌ 不可对 UUID 做大小写转换
```

✅ **正确的做法**：

```
output/results/81B39041-0840-4C89-9F4B-075A80F2CDD4/  (上游如果是UUID，保持UUID格式)
output/results/219A8C6D8C334629A7E1F164D514C381/      (上游如果是大写，保持大写)
```

### 3. 本地持久化文件规范

#### 3.1 必须创建的三个文件

质检完成后，**必须在上述目录中创建以下三个文件**，不可遗漏任何一个：

| 文件类型           | 文件名规范                                   | 内容                                                   | 必需 |
| ------------------ | -------------------------------------------- | ------------------------------------------------------ | ---- |
| **完整结果** | `{timestamp}_{task_id}.complete.json`      | 完整的质检结果，包含所有维度的详细检查结果和证据       | ✅   |
| **摘要结果** | `{timestamp}_{task_id}.summary.json`       | 精简版本，仅包含关键字段（status、score、explanation） | ✅   |
| **结果索引** | `{timestamp}_{task_id}.results_index.json` | 索引文件，记录所有已保存结果的元数据和文件位置         | ✅   |

任一必需文件写入失败时，本次持久化必须视为失败，不可继续回库。

#### 3.2 时间戳格式

时间戳格式为：`YYYYMMDD_HHmmss`

| 格式 | 说明             | 示例 |
| ---- | ---------------- | ---- |
| YYYY | 4位年份          | 2026 |
| MM   | 2位月份（01-12） | 03   |
| DD   | 2位日期（01-31） | 06   |
| HH   | 2位小时（00-23） | 10   |
| mm   | 2位分钟（00-59） | 30   |
| ss   | 2位秒钟（00-59） | 45   |

**完整示例**：

```
20260306_103045_219A8C6D8C334629A7E1F164D514C381.complete.json
20260306_103045_219A8C6D8C334629A7E1F164D514C381.summary.json
20260306_103045_219A8C6D8C334629A7E1F164D514C381.results_index.json
```

#### 3.3 文件内容要求

**complete.json**：

- 包含完整的 `qc_result` 对象（符合 `schema/qc_result.schema.json`）
- 包含所有维度的详细检查结果
- 包含完整的证据数据和匹配信息
- 不可进行任何数据删除或简化

**summary.json**：

- 精简版本，仅保留关键字段
- 包含：task_id、qc_status、qc_score、has_risk、explanation
- 包含各维度的 status（pass/risk/fail）
- 适合快速查询和统计

**results_index.json**：

- 自动维护的元数据索引
- 记录所有已保存结果的位置和元数据
- 支持快速定位文件
- 格式详见：`SKILL.md` 的“本地持久化要求”部分

## 📤 回库约束

### 4. 质检结果回库

#### 4.1 禁止编写回库脚本

❌ **严格禁止**：

- 自行编写 Python 脚本来处理回库逻辑
- 创建临时的数据库连接和更新脚本
- 硬编码 SQL 语句进行数据更新
- 绕过现有的回库技能直接操作数据库

✅ **必须使用**：

- 调用现有的回库技能：`qc-write-pg-qc` (v1.1.0)
- 该技能位于：`../qc-write-pg-qc/`
- 技能文档：`../qc-write-pg-qc/SKILL.md`

#### 4.2 回库技能调用方式

**Python 调用**：

```python
from qc_write_pg_qc.SKILL import execute

result = execute({
    'task_id': '219A8C6D8C334629A7E1F164D514C381',
    'result_dir': 'output/results',
    'table_name': 'poi_qc_zk'  # 可选，默认为 poi_qc_zk
})
```

**命令行调用**：

```bash
python ../qc-write-pg-qc/SKILL.py 219A8C6D8C334629A7E1F164D514C381 output/results poi_qc_zk
```

#### 4.3 回库技能特性

| 特性               | 说明                               |
| ------------------ | ---------------------------------- |
| **灵活表名** | 支持指定不同的目标表（v1.1.0新增） |
| **容错机制** | 索引文件缺失时自动扫描目录定位文件 |
| **原子性**   | 事务处理确保数据一致性             |
| **幂等性**   | 重复执行不会产生重复记录           |
| **安全性**   | 表名经过SQL注入防护验证            |

#### 4.4 回库流程

```
质检完成
   ↓
生成三个本地文件 ← 【检查点】必须包含 complete.json、summary.json、results_index.json
   ↓
调用 qc-write-pg-qc 技能
   ↓
技能自动：
  1. 从结果目录读取文件
  2. 解析质检结果
  3. 数据格式转换
  4. 写入PostgreSQL
   ↓
返回执行结果
```

## ⚠️ 违反约束的后果

如果违反了以上约束条件：

| 违反项目                 | 后果                                       |
| ------------------------ | ------------------------------------------ |
| 编写了Python脚本         | 质检结果无效，需要重新执行                 |
| 文件夹不是 task_id 命名  | 回库技能无法定位文件，回库失败             |
| 文件名没有时间戳         | 索引文件无法正确维护，历史追踪困难         |
| 缺少三个文件中的任何一个 | 质检结果不完整，无法通过校验               |
| 自写回库脚本             | 绕过了数据安全检查，可能导致数据一致性问题 |
